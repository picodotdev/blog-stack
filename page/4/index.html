Por favor, comenta el código

Escrito por Juan María Hernández en la bitácora Koalite, artículo original
Publicado el lunes, 07 de julio de 2014 a las 05:06 UTC
Etiquetado como development, software-development

Recuerdo que cuando empecé a programar en la carrera (hace ya unos cuantos años), una de las cosas en la que más nos insistían los profesores era en que usásemos comentarios en nuestro código. Muchos comentarios. Prácticamente uno por cada línea. Visto con perspectiva, es posible que sólo trataran de averiguar la intención que había detrás de ese código tan enrevesado que escribíamos para cualquier cosa, o de obligarnos a pensar lo que queríamos hacer antes de empezar a escribir cada línea de código.

El hecho es que esa filosofía de sobreabundancia de comentarios llevaba escribir código de este estilo:

// suma los elementos de un vector
// param: vector con los datos a sumar
// return: suma de los elementos
int sumaVector(int[] vector) {
  // Variable para almacenar la suma de los elementos
  int suma = 0; 
  // Recorremos el vector 
  for (int i = 0; i < vector.length; i++) {
    // Obtenemos el elemento i-ésimo del vector
    int elem = vector[i];
    // Actualizamos la suma
    suma = suma + elem;
  }
  return suma;
}

Aquel que me conozca sabrá que tengo un punto bastante pedante-busca-palabras. Necesito verbalizarlo todo. Muchas veces discurro inventándome diálogos conmigo mismo o como si le explicara las cosas a otro… (sí, estoy un poco pa´allá… :-P). El caso es que me doy cuenta mientras hablo de que muchas veces confundo simple con fácil o complicado con complejo. Independientemente de lo tiquismiquis que uno se quiera poner con el idioma, creo que eso tiene consecuencias a la hora de entender a qué estamos cuando trabajamos. Pero primero, veamos la definición de cada término:

  • fácil: que cuesta poco trabajo. Que puede ocurrir con mucha probabilidad.
  • difícil: que no se logra sin mucho trabajo.
  • simple: sin complicación. Formado por un solo elemento o por pocos. De fácil comprensión.

Stress

Publicado el martes, 01 de julio de 2014 a las 07:17 UTC
Etiquetado como desarrollo-de-software

Entresacando lo que me da la gana de la wikipedia:

  • El estrés es nerviosismo o inquietud.
  • Una respuesta natural y necesaria para la superviviencia ante un agente externo.
  • Es sólo patología cuando se produce una sobrecarga de tensión que produce enfermedades y anomalías como problemas de memoria, de concentración y de ánimo.

No se puede decir que yo sea una persona que viva muy estresada. Éste año me está tocando meter horas porque hace falta y, como todo el mundo, cuando sobretrabajas, te agotas mentalmente y tu cuerpo se resiente. En mi caso, me suele llevar a soñar con código, no poder desconectar, comer con ansiedad y engordar, etc. Pese a todo y en general, no tiene mayores consecuencias. Sin embargo, en mi círculo personal de amigos y familiares he podido ver un par de casos concretos de lo que el estrés le puede hacer a la gente, que me han hecho estar más alerta con éste tema.
