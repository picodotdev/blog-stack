El sistema operativo libre similar a UNIX, OpenBSD, acaba de lanzar la nueva versión 5.5

Sobre OpenBSD

OpenBSD es un sistema operativo libre liberado bajo la licencia BSD, desciende del sistema  NetBSD. Prioriza la seguridad y la criptografía. Su kernel es de tipo monolítico.


Novedades

  • Escritorios: GNOME 3.10.2 ,KDE 3.5.10 y 4.11.5 y Xfce 4.10
  • Soporte 64 bits para time_t en todas las plataformas
  • Bases de datos: MySQL 1.5.73 y PostgreSQL 9.3.2
  • Sistema de correo: Postfix 2.11.0
  • Versión de LDAP 2.3.43 y 2.4.38
  • Internet: Mozilla Firefox 24.3 y 26.0
  • Paquete ofimático: LibreOffice 4.1.4.2
  • Desarrollo: PHP 5.3.28 y 5.4.24
  • Python 2.7.6 y 3.3.2
  • GCC 4.6.4 y 4.8.2

Listar los usuarios de un sistema GNU/Linux

Actualizado el viernes, 25 de abril de 2014 a las 05:59 UTC
Etiquetado como linux, sistemas-operativos, comandos, uid-number, passwd, etc, gnu, cat

Un administrador de sistemas tiene que controlar decenas de máquinas, con sus respectivos usuarios. Eso puede ser una tarea bastante farragosa, aunque lleves un control de base de datos o en una simple hoja de cálculo, con todos los usuarios.

Una manera fácil de ver todos los usuarios es mediante la visualización del archivo passwd, ubicado en la carpeta etc.

Lo haríamos de la siguiente manera:

cat /etc/passwd

Si tenemos una lista podemos usar un filtro, con el comando grep

cat /etc/passwd | grep "busqueda"

Analizamos un poco los resultados que suelen dar:

Listado de passwd

Destripemos los resultados:

  • Login ID. Que se es el nombre de la cuenta.
  • x. Vemos x en representación del password encriptado.
  • La siguiente cifra es el número de identificación del usuario o UID number. Ampliando la información podemos decir que éste número es realmente con el que trata el sistema en sus procesos, para facilitarle las cosas.

La nueva versión de Apache soluciona fallos de seguridad

Escrito por ochobitshacenunbyte en la bitácora Ocho bits hacen un byte, artículo original
Publicado el viernes, 21 de marzo de 2014 a las 12:43 UTC
Actualizado el viernes, 21 de marzo de 2014 a las 12:54 UTC
Etiquetado como apache, comunidad-open-source, sistemas-operativos, dos, desarrollo, http

El famoso software de código abierto Apache HTTP Server, ha lanzado la nueva versión 2.4.9, donde corrige dos problemas graves de seguridad.

El primer de ellos soluciona un fallo que podría dar lugar una ataque de denegación de servicio (DoS). Dicho error ha sido clasificado por el “National Vulnerability Database” de la siguiente manera (traducido del inglés) :

“La función log_cookie en mod_log_config.c en el módulo mod_log_config, en el Servidor Apache HTTP antes de la 2.4.8 (anterior versión) permite a atacantes remotos provocar una denegación de servicio (fallo de segmentación y caída del demonio) a través de una galleta hecha a mano que no está controlada adecuadamente durante el truncamiento.“

La solución a esta vulnerabilidad desde la nueva versión de Apache, hace que el servidor limpie el registro de cookies de forma que ya no reconoce las cookies que no valen.

Publicado el jueves, 06 de marzo de 2014 a las 08:42 UTC
Actualizado el jueves, 06 de marzo de 2014 a las 08:52 UTC
Etiquetado como centos, postfix, sistemas-operativos, gnu-linux, rpm, chkconfig, fedora, opensuse

Una de las cosas que más me trajo de cabeza las primeras veces que traté con servidores con OpenSuse, en su diferentes versiones, es que al reiniciar, los servicios que había instalado no se activaban al inicio, cosa que si hacían los sistemas basados en Debian.

La clave la tiene el comando chkconfig y qué los sistemas basados en paquetes rpm, como OpenSuse o CentOS o Fedora no configuren los servicios recientemente instalados en al arranque del sistema.

Si suponemos que hemos instalados, por ejemplo, el servicio de correo POSTFIX, para que se inicie en el arranque lo tenemos que hacer de la siguiente manera:

chkconfig postfix on

Si antes queremos comprobar si el servicio está configurado en el arranque, simplemente:
