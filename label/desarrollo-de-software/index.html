Creación de objetos eficiente en JavaScript

Escrito por Guille en la bitácora Cuaderno de software, entrada original
Publicado el viernes, 02 de noviembre de 2012 a las 12:17 UTC
Etiquetado como prototipos, desarrollo-de-software, javascript

There’s more than one way to skin a cat

– Proverbio inglés.

Hay muchas maneras de crear objetos en JavaScript y la que elijamos debe depender de las necesidades particulares de cada problema, pero siempre tenemos que tener en mente la eficiencia del código que escribamos.

En general, queremos que nuestro código fuente se ejecute una sola vez y que prepare en memoria todo lo necesario para generar objetos, uno detrás de otro, como churros y con el mínimo esfuerzo posible. Por ello, la mejor opción consiste en usar una función constructora y añadir aquello que no sea el estado de una nueva instancia en el prototipo de la función constructora:

var Counter = function(offset) {
  this.value = offset || 0;
};
 
Counter.prototype.increase = function() {
  return ++this.value;
};
 
Counter.prototype.decrease = function() {
  return --this.value;
};
