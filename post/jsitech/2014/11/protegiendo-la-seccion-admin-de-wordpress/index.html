Protegiendo la Sección Admin de WordPress

Escrito por Jason Soto en la bitácora JsiTech, artículo original
Publicado el domingo, 30 de noviembre de 2014 a las 22:40 UTC
Etiquetado como seguridad, cms, tutoriales, software-libre, wordpress

Si hay algo que he notado es que no protegemos lo suficiente la sección de Admin de los que usamos WordPress o Cualquier otro CMS. Decidi ponerme la tarea de ver la página de admin de algunos de los sitios de WordPress que sigo y en la mayoría solo he visto que tiene un plugin contra los intentos fallidos. Pero, y si alguien logra obtener los datos de acceso? pues ya no hay nada que podamos hacer. Entiendo que a veces pensamos, solo uso wordpress para bloguear, no creo que nadie quiera intentar hacer nada ya que no hay nada importante ahí, pero lo cierto es que hasta para hacernos algún tipo de daño un usuario malintencionado puede intentar algo y es por eso que entiendo que debemos ponerle una capa extra de protección. Como se trata tal vez de un sitio que tenemos para informaciones no es necesario que coloquemos ahi un certificado digital pero si podemos protegerlo con autenticacion de 2 factores.

Hoy les voy a hablar de 2 herramientas que nos pueden ayudar en agregar una capa extra de seguridad, Google Authenticator y Latch de Eleven Path.

Antes de iniciar a hablar de estas herramientas aqui les dejo una captura de pantalla donde obtuve los datos de acceso de mi WordPress, claro, simplemente llevando a cabo un ataque de hombre en el medio MITM, estando en la misma red, y se que en este caso se preguntarán, pero lo hiciste conectado a la misma red, claro, pero se que muchos utilizan las redes inalambricas públicas para crear una entrada rápida o ver algo en la sección admin, lo cierto es que no sabemos las intenciones de los otros usuarios que se encuentran en esa red.

Google Authenticator es un software basado en autenticación de contraseñas de un solo uso desarrollado por Google, que ofrece un número de seis digitos que cambia de manera aleatoria, que el usuario debe suministrar ademas de su usuario y contraseña para acceder a las cuentas que soporta la aplicación, entre estas podemos mencionar, las cuentas de google, correo de microsoft, Dropbox, WordPress, Joomla, entre muchos otros servicios. Incluye implementaciones para la generación de contraseñas destinados a varias plataformas móviles, así como un módulo de autenticación conectable PAM. Los códigos son generados haciendo uso del standards abiertos desarrollados por The Initiative for Open Authentication OATH.

Google Authenticator tiene aplicación móvil para Android, IOS y Blackberry, solamente deben descargarla desde la tienda de aplicaciones correspondiente.

Para proteger nuestro WordPress basta con descargar un Plugin llamado Google Authenticator. Una vez instalado solo tenemos que ir a nuestro perfil y escanear el código de barra para sincronizarlo con la aplicación de nuestro telefono.

Una vez completemos este paso, la próxima vez que vayamos a acceder al admin de nuestro WordPress nos pedirá el código generado por la aplicación de Google Authenticator.

Si bien ese código se podría captar por igual que la contraseña, no le funcionará ya que son de uso único y tendrían que utilizar el próximo que solo estará disponible para ser visto en su teléfono.

Otra Herramienta que me parece muy util, es Latch.

Latch le permite agregar otra capa de seguridad a nuestra sección de admin de WordPress, se encarga de desactivar el acceso cuando no lo estamos utilizando minimizando el riesgo de cualquier acceso no autorizado.

Para utilizar en nuestro WordPress simplemente descargamos el Plugin para WordPress Latch de Eleven Path, una vez hecho esto accedemos al portal de Eleven Path y nos registramos como Developers.

En este paso podemos agregar los sites a proteger desde nuestro Smartphone descargando la aplicación de Latch desde la tienda de aplicaciones o desde el mismo portal. Simplemente seleccionamos que deseamos agregar un servicio nuevo y este nos generará un Token que debemos ingresar en nuestro perfil de WordPress en la sección que dice Latch Token. Una vez sincronizados podremos desactivar y activar el acceso desde el portal de Latch o desde la aplicación.

No mostré todos los pasos al detalle ya que en mi próximo Hangout voy a mostrar en vivo, paso a paso,  como implementamos estas capas extras de seguridad en WordPress. Estaré creando una entrada con los detalles.
