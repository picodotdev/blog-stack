AIDE y como instalarlo en Linux CentOS/RHEL/Fedora

Escrito por Jason Soto en la bitácora JsiTech, artículo original
Publicado el lunes, 17 de noviembre de 2014 a las 22:48 UTC
Etiquetado como ids, seguridad, aide, software-libre, linux

Uno de los indicadores donde nos podemos dar cuenta de que alguien ha comprometido nuestro sistema es cuando detectamos algún cambio en nuestros archivos o archivos que no conocemos, pero como detectamos esto? al menos que revisemos todos los archivos dia a dia que sería algo bastante trabajoso, lo más seguro es que necesitemos una herramienta que haga esas verificaciones y nos alerte cuando detecte un cambio, y es aquí donde entra AIDE ( Advanced Intrusion Detection Environment ) Ambiente de detección de intrusos avanzada.

AIDE crea una base de datos de las expresiones regulares que encuentra en los archivos de configuración. Una vez se inicializa la base datos puede ser utilizado para verificar la integridad de los archivos, los atributos tambien pueden ser verificados por inconsistencia y puede leer la base de datos de versiones anteriores o mas actualizadas. Utiliza una serie de algoritmos de verificación de integridad y cuenta con varias características que mencionamos:

  • Algoritmos soportados: md5, sha1, rmd160, tiger, sha256, sha512, whirlpool
  • Atributos de archivos soportados: Tipos de archivo, permisos, Inode, UID, GID, Tamaño, Conteo de bloque, Nombre de enlace, cantidad de enlaces, entre otros.
  • Archivo de configuración y base de datos en texto plano para simplicidad
  • Soporte para compresión gzip de la base de datos
  • Capacidad para excluir o incluir archivos y directorios para ser monitoreados
  • entre otros,

Vamos a ver como lo instalamos en nuestro servidor para que monitoreemos todo el tiempo la integridad de nuestros archivos.

Vamos a seguir estos pasos como root.

#yum install aide

Procedemos a crear la base de datos inicial, es recomendable que se cree en un estado limpio del sistema.

#aide –init

debemos renombrar la base de datos para que concuerde con el archivo de configuración /etc/aide.conf

# cp /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz

Vamos a correr nuestra primera verificación

# aide –check

Vemos que nos devuelve que todo esta bien

AIDE, version 0.14

### All files match AIDE database. Looks okay!

Vamos a crear un archivo a ver que nos dice

# touch prueba

Podemos ver que nos notifica que hay un archivo nuevo.

Si queremos podemos actualizar la base de datos con los cambios.

# aide –update

Si hacemos un cambio en un archivo veremos algo como esto. En este caso modifique /etc/sysconfig/network

Vemos que nos da detalles de los cambios en el tamaño del archivo y fecha del cambio.

Para automatizar un poco, vamos a crear una tarea cron que corra aide diariamente y nos envie un correo con los resultados, de esta forma podemos tomar acción en caso de que detectemos un cambio no autorizado.

#crontab -e

0 0 * * * aide –check | sendmail -s ” Chequeo Diario de integridad AIDE $HOSTNAME” sucorreo@correo.com

Entiendo que una de las mejores prácticas de seguridad es contar con este sistema de detección para estar al tanto de nuestro sistema y detectar cualquier cambio no autorizado. Hasta aquí lo dejo, mas adelante subo el tutorial para Distros con Debian/Ubuntu. Estamos ademas integrando la instalacion de AIDE al deployer de seguridad JackTheStripper.
