Vulnerabilidad de Seguridad Importante en Bash

Escrito por Jason Soto en la bitácora JsiTech, artículo original
Publicado el jueves, 25 de septiembre de 2014 a las 01:55 UTC
Etiquetado como linux, bash, shell, noticias, seguridad

Se ha dado a conocer una vulnerabilidad importante en el Shell de Linux, Bash. Como Todos conocemos las variables de entorno en Linux es una forma de modificar el comportamiento de una aplicación en nuestro sistema. Usualmente consiste en unos valores que le asignamos a esos entornos. Asi mismo sucede con Shell, son muchos los programas que llaman al shell en el fondo para correr algunas instrucciones o dar algunos servicios. Para mencionarles algunos ejemplos, ssh llama al shell para mostrarlo a un usuario remoto, apache puede llamarlo para correr un script o dar soporte a la ejecución de algún comando limitado.

La Vulnerabilidad surge por el hecho de que se pueden crear variables de entorno con valores especialmente diseñados antes de llamar el shell bash. Estas variables pueden contener código arbitrario que pueden ser ejecutado cuando el shell es invocado. El resultado es que esta vulnerabilidad puede ser expuesta en muchos contextos:

  • ForceCommand es usado en sshd para proveer la capacidad de ejecutar algunos comandos limitados a usuarios remotos, esta falla puede ser usada para burlarlo y permitir la ejecución de código arbitrario. Algunas implementaciones de Git y Subversion usan estos shells restringidos.
  • Servidores de Apache que usan mod_cgi o mod_cgid son afectados si los scripts estan escritos en Bash o invocan un shell. Estos shells son implicitamente usados por system/popen en C, por os.system/os.popen en Python, system/exec en PHP cuando corre en modo CGI y open/system en Perl.
  • Scripts PHP ejecutados con mod_php no son afectados aun invoquen un Shell.
  • Clientes DHCP que invocan un shell para configurar el sistema, con valores tomados de un potencial servidor malicioso, esto permitiría la ejecución de código arbitrario, típicamente como root, en el cliente DHCP.
  • Muchos Daemons y programas privilegiados pueden ejecutar shell scripts con valores de variables de entorno configurados por el usuario que pueden permitir ejecución de código arbitrario
  • Cualquier aplicación enlazado al shell o que corre scripts usando bash como intérprete.

Como cualquier lenguaje de programación, Bash tiene funciones en una implementación limitada, y es posible colocar estas funciones en variables de entorno. Esta falla es expuesta cuando se ingresa código extra en la definición de estas funciones, dentro de las variables de entorno.

Ya los paquetes para solucionar están disponibles en la mayoría de distribuciones, asi que se recomienda que se actualice al sistema inmediátamente para mitigar los riesgos.
