Avisos de Seguridad 20/09/2014

Escrito por Jason Soto en la bitácora JsiTech, artículo original
Publicado el sábado, 20 de septiembre de 2014 a las 15:30 UTC
Etiquetado como linux, avisos, software-libre, seguridad

UBUNTU

Ubuntu USN-2349-1

Paquete: libav (Player Multimedia, servidor, codificador)

Versiones afectadas: 12.04 LTS

Se notificó que el paquete libav maneja de forma incorrecta ciertos archivos multimedia. Si un usuario es engañado para abrir un archivo especial multimedia, un atacante puede causar una denegación de servicio mediante un cierre inesperado de la aplicación o ejecutar código arbitrario con los permisos del usuario que lo ejecutó.

El problema fue corregido en la versiones,

libavcodec53 4:0.8.16-0ubuntu.12.01.1

libavformat53 4:0.8.16-0ubuntu.12.01.1

Ubuntu USN-2347-1

Paquete: Python-django

Versiones afectadas: 14.04 LTS, 12.04 LTS. 10.04 LTS

Múltiples Vulnerabilidades encontradas:

  • Django valida incorrectamente las URL’s, un atacante puede llevar a cabo un ataque de Phishing
  • Django maneja de manera incorrecta la generación de nombres de archivo, un atacante remoto puede causar que django consuma muchos recursos, resultando en un ataque de Denegación de Servicio
  • Django maneja de forma incorrecta algunos headers en contrib.auth.middleware.RemoteUserMiddleware, esto le permite a un atacante robar una sesión web

Estas vulnerabilidades pueden ser solucionadas actualizando los paquetes a la siguiente versión.

Ubuntu 14.04 LTS

python-django 1.6.1-2ubuntu0.4

Ubuntu 12.04 LTS

python-django 1.3.1-4ubuntu1.12

Ubuntu 10.04 LTS

python-django 1.1.1-2ubuntu1.13

Se recomienda una actualización Completa del sistema.

Ubuntu USN-2319-3

Paquete: openjdk-7

Versiones afectadas: Ubuntu 14.04 LTS

Multiples vulnerabilidades encontradas en OpenJDK JRE referente a divulgación de Información, disponibilidad e integridad de los datos, le permite a un atacante explotarla para causar un ataque de denegación de Servicio o exponer informaciones sensible por la red.

Este problema se corrige en las siguientes versiones del paquete:

Ubuntu 14.04 LTS

icedtea-7-jre-jamvm        7u65-2.5.2-3~14.04

openjdk-7-jre                    7u65-2.5.2-3~14.04

openjdk-7-jre-headless   7u65-2.5.2-3~14.04

openjdk-7-jre-lib               7u65-2.5.2-3~14.04

openjdk-7-jre-zero            7u65-2.5.2-3~14.04

Ubuntu USN-2327-1

Paquete: squid3

Versiones Afectadas: 14.04 LTS, 12.04 LTS

Vulnerabilidad encontrada en el paquete squid3 que le permite a un atacante inhabilitar el servicio al recibir tráfico de red modificado al no poder analizar correctamente la validación de la solicitud.

Se recomienda actualizar el sistema,

Ubuntu 14.04 LTS

squid3          3.3.8-1ubuntu6.1

Ubuntu 12.04 LTS

squid3          3.1.19-1ubuntu.12.04.3

Para actualizar todo el sistema y solucionar las vulnerabilidades antes mencionada 

# apt-get update 

# apt-get upgrade

FEDORA

Fedora 2014-10632

Paquete: pdns-recursor (moderno y avanzado servidor de nombres)

Fedora 20

Vulnerabilidad: Una secuencia de paquete específica puede inhabilitar el servicios de PowerDNS Recursor 3.6.0 de manera remota

Se recomienda actualizar con yum update

Fedora 2014-7570 7551

Paquete: Asterisk

Fedora 19/20

Actualización de Seguridad para solucionar múltiples vulnerabilidades en asterisk que le permiten a un atacante llevar a cabo un ataque de Denegacion de Servicio y la posibilidad de una elevacion de privilegios en el sistema victima.

Actualizar con Yum update

Fedora 2014-10322

Paquete: apache-poi

Fedora 20

Actualización de Seguridad para el Paquete apache-poi para solucionar Múltiples Vulnerabilidades que le permiten a un atacante acceder a los documentos del usuario analizados por apache-poi, y llevar ataques mas avanzados como denegación de servicios mediante un consumo excesivo de cpu y memoria.

Actualizar a la versión 3.10.1 con yum update.

OPENSUSE

OpenSUSE-SU-2014:1144-1

Paquete: squid

Versiones Afectadas: OpenSUSE 13.1/12.3

Actualización de Seguridad para solucionar vulnerabilidad que le permite a un atacante llevar a cabo un ataque de Denegación de Servicios inhabilitando el servicio de squid.

Actualizar

zypper in -t patch openSUSE-2014-548

 

openSUSE 13.1

squid-3.3.13-2.10.1

squid-debuginfo-3.3.13-2.10.1

squid-debugsource-3.3.13-2.10.1

openSUSE 12.3

squid-3.2.11-3.16.1

squid-debuginfo-3.2.11-3.16.1

squid-debugsource-3.2.11-3.16.1

CENTOS

CESA-2014-1148

Paquete: squid3/squid2

Versione afectadas: todas

Vulnerabilidad encontrada en el paquete squid3 no valida de forma correcta la entrada en un análisis de solicitud, esto le permite a un atacante llevar a cabo un ataque de Denegación de Servicios al enviar solicitudes modificadas.

Se recomienda actualizar sistema,

# yum update

DEBIAN

DSA-3014-1

Paquete: squid3

Vulnerabilidad encontrada en el paquete squid3 no valida de forma correcta la entrada en un análisis de solicitud, esto le permite a un atacante llevar a cabo un ataque de Denegación de Servicios al enviar solicitudes modificadas.

Esta vulnerabilidad fue solucionada en:

Para la versión estable (Wheezy), versión 3.1.20-2.2+deb7u2

Para la versión inestable (sid), sera solucionado próximamente
