Monitoreando Infraestructura con Zabbix 2.4

Escrito por Jason Soto en la bitácora JsiTech, artículo original
Publicado el sábado, 11 de octubre de 2014 a las 22:50 UTC
Etiquetado como zabbix, centos, linux, tutoriales, software-libre

Hoy en día es de vital importancia que monitoreemos nuestra infraestructura para reducir al minimo cualquier tiempo de inactividad. Mientras mas grande es nuestra infraestructura, es un poco más dificil ver el estado de cada uno de los equipos si no contamos con una solución centralizada que nos ayude con este monitoreo. Hay muchas soluciones que nos pueden ayudar con esta tarea y para mencionarla algunas, tenemos a Nagios, Cacti, Zenoss, Zabbix que es del que les voy a hablar en este artículo. Cada uno tiene sus diferencia, pero los tocaré ya en otro artículo.

Zabbix es un software de monitoreo de infraestructura de código Libre, capaz de monitorear toda una infraestructura en una organización, ofrece monitoreo de servidores, equipos de comunicaciones, aplicaciones y más. Todo esto lo hace por una agente o si deseamos lo podemos hacer usando el protocolo SNMP. Pueden ver mas informaciones de la solución Aquí.

Vamos a ver ahora como podemos implementar la solución en un servidor con Linux Centos, y para eso seguimos estos pasos como el usuario root.

Instalamos los Prerequisitos

# yum install mysql mysql-server

# yum install httpd

# yum install php php-cli php-common php-devel php-pear php-gd php-mbstring php-mysql php-xml

Subimos los servicios de apache y MySQL y hacemos la configuración inicial de mysql

# chkconfig –level 35 httpd on

# chkconfig –level 35 mysqld on

# service httpd start

# service mysqld start

# mysql_secure_installation

Este último comando tiene como objetivo hacer una configuración inicial segura de mysql, aquí simplemente la asignamos una contraseña a root y seleccionamos los valores por defecto.

Finalizado esto, ahora vamos con la instalación de Zabbix 2.4

Instalamos el Repo de Zabbix

# rpm -Uvh http://repo.zabbix.com/zabbix/2.4/rhel/6/x86_64/zabbix-release-2.4-1.el6.noarch.rpm

Instalamos Zabbix Server

# yum install zabbix-server-mysql zabbix-web-mysql zabbix-agent zabbix-javagateway

Modificamos la zona horaria en el archivo php.ini

# vi /etc/php.ini

date.timezone = America/Santo_Domingo

Esto es un ejemplo, para confirmar como configurar su zona horaria pueden consultar, http://php.net/manual/en/timezones.php

Una vez configurado guardamos, salimos y reiniciamos el servicio de Apache.

# service httpd restart

Creamos la Base de datos de Zabbix y corremos los scripts de zabbix

# mysql -u root -p

mysql> create database zabbix;

mysql> grant all privileges on zabbix.* to ‘zabbix’@’localhost’ identified by ‘Contraseña';

mysql> flush privileges

mysql> use zabbix;

mysql> source /usr/share/doc/zabbix/zabbix-server-mysql-2.4.0/create/schema.sql

mysql> source /usr/share/doc/zabbix/zabbix-server-mysql-2.4.0/create/images.sql

mysql> source /usr/share/doc/zabbix/zabbix-server-mysql-2.4.0/create/data.sql

mysql> exit;

Iniciamos el servicio de Zabbix

# chkconfig –level 35 zabbix-server on

# chkconfig –level 35 zabbix-agent on

# service zabbix-server on

# service zabbix-agent on

Solo nos falta configurar el servidor por el ambiente gráfico, para esto accedemos a el por un navegador via:

http://IP-Servidor/zabbix/setup.php

y seguimos las instrucciones ingresando la información de la base de datos que creamos anteriormente.

Luego de finalizado, para ingresar a la interfaz lo haremos por las credenciales por defecto

Usuario: admin

contraseña: zabbix

En otros artículos veremos como podemos agregar los hosts e ir configurando los templates de monitoreo y alertas, mientras tanto pueden ir conociendo el sistema.
