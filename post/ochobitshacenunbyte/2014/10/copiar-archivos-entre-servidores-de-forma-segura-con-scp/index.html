Copiar archivos entre servidores de forma segura con SCP

Escrito por ochobitshacenunbyte en la bitácora Ocho bits hacen un byte, artículo original
Actualizado el martes, 28 de octubre de 2014 a las 16:26 UTC
Etiquetado como blog-stack, ssh, sistemas-operativos, scp, comandos

Hoy hablaremos de una herramienta muy común para los administradores de sistemas GNU/Linux, pero que hasta ahora no había tratado en la web. Se trata de SCP.

SCP utiliza el protocolo que lleva el mismo nombre o Secure Copy para realizar transferencias de archivo entre servidores, un host local y otro remoto. Utiliza el protocolo Secure Shell o SSH

El envío de los datos son cifrados, para que así no se pueda extraer información de los paquetes si son interceptados.

Su funcionamiento no es muy complicado, la sintaxis sería la siguiente:

scp usuario@host:directorio/ArchivoOrigen ArchivoDestino

O a la inversa:

scp ArchivoOrigen usuario@host:directorio/ArchivoDestino

El programa SCP utiliza el mismo puerto configurado para SSH.

Ya hablé en un artículo anterior cómo cambiar el puerto por defecto de SSH.

Así que si no es el 22, debemos de indicar al programa el puerto indicado.

Para poder realizar utilizaremos el parámetro -P

scp -P [puerto] usuario@host:directorio/ArchivoOrigen ArchivoDestino

Para saber más podéis consultar la página web de wikipedia, en castellano.
