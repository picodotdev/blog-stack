Utilizando vagrant para tener entornos de pruebas (LAMP)

Escrito por David González en la bitácora Cuaderno de software, artículo original
Publicado el domingo, 09 de marzo de 2014 a las 18:55 UTC
Etiquetado como vagrant, lamp, gestion-de-infraestructura, integracion-continua

Muchas veces nos vemos en la necesidad de disponer de diferentes configuraciones (versiones de software, configuraciones de seguridad, librerias no disponibles en todos los sistemas operativos…) o de un entorno lo mas parecido al de producción, lejos de la configuración de nuestra máquina de desarrollo. Las máquinas virtuales son una solución bastante versatil para esto, pero muchas veces andar instalando el sistemas base, configurando directorios compartidos y demás resulta muy pesado.

Vagrant es un manejador de máquinas virtuales que nos facilita esta tarea. En este screencast veremos una pequeña aproximación utilizando como ejemplo es stack LAMP.
