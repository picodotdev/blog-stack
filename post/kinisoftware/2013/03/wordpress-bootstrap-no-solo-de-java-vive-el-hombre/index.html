WordPress + Bootstrap, no solo de Java vive el hombre

Escrito por kinisoftware en la bitácora Kinisoftware, artículo original
Publicado el lunes, 04 de marzo de 2013 a las 08:28 UTC
Etiquetado como web, gente-interesante, wordpress, herramientas

Lo confieso, estoy enamorado de WordPress y Bootstrap. Y, si los juntamos, ya no os digo ná La verdad es que mi relación con WordPress siempre ha sido muy “básica”, para montar un sitio Web de forma rápida, sencilla y funcional desde el minuto 0. Pero desde el último mes he estado involucrado en un proyecto muy chulo de unas amigas (luego haré promoción) que me ha llevado a conocer WordPress algo mejor. Además ha dado la casualidad que gracias a mis KotaÑeros (Fran y Dani rock!) de Cáceres, unos cracks de la parte front, me enseñaron (y obligaron a usar) Bootstrap y aquello ya no tenía vuelta atrás. <3

Todo empezó cuando @en_voz_baja me habló de un proyecto que tenía en la cabeza y tenía toda la ilusión del mundo por sacarlo a luz. Hoy ese proyecto se conoce como Reparando Alas Rotas y me encanta desde el nombre hasta el último de sus productos. Mi recomendación es que os paséis por la Web (que he hecho yo y ahora os explicaré como) y os empapeis de la ilusión y espíritu de este par de reparadora de alas rotas

La Web actual del proyecto no es “una cosa mega-innovadora” y “técnicamente espectacular”. No tenía porque serlo. He procurado ser práctico e irme a cosas sencillas pero, a la vez, que supusiera un reto, como aprender “eso de Bootstrap” o “el responsive design de los w…” Aún le faltan cosas que mejorar, hay “work in progress” y a mi mucho por aprender pero creo que no ha quedado mal para ser “la primera que hago”. Además, ¡lo importante es el contenido!

Básicamente es WordPress (con algunos plugins) y Bootstrap, con un tema propio creado por mi para la ocasión. Al principio creé HTML estático con lo que quería (que cambiaba bastante según las “chicas reparadoras” me daban feedback)  y luego he troceado y volcado ese HTML a lo que sería la composición de un tema WordPress. Desde el principio usé Bootstrap y me ha facilitado la vida muchísimo aparte de hacer que me olvide de problemas de responsive o css que ya tiene resueltos. Para construir el tema a partir del HTML estático seguí un artículo que encontré de los chicos de TreeHouse que aquí os pongo. Con eso hice lo más gordo. Luego quedaba el 20% que se llevó el 80% del tiempo Retocar fuentes, posicionamiento de elementos, comprobar el responsive en tablet, móvil, tamaño de pantalla, etc.

Lo he subido a Bitbucket (quería probar el servicio porque GitHub ya lo conozco) y he aprovechado también las opciones de reportar Issues para ir registrando (y que las chicas reparadoras también puedan) tareas, bugs, mejoras, etc. Aquí os dejo el enlace, realmente no sé si será muy útil pero está muy “anclado” a este proyecto y tengo imágenes y recursos que “creo entender” que no deberían ir ahí pero bueno ^_^

Como conclusión saco que WordPress está muy muy currado, sé nota la experiencia, sobre todo se nota que los casos más habituales están solucionados de forma sencilla y creo que con buenas decisiones. Cuesta muy poco hacerte un tema o aprovechar sus capacidades. Además Bootstrap me parece esencial (o algún framework similar) para hacer un desarrollo front-end hoy en día. Quizás sea por mi “incapacidad de parte cliente” pero me ha encantado lo que he visto. Desde aquí un EPICWIN para ambos ¡Son unos de mis nuevos mejores amigos!
