Solucionar el error de Vidalia + TOR en Ubuntu

Escrito por Dabid Martinez en la bitácora Laborategia, artículo original
Publicado el jueves, 16 de octubre de 2014 a las 06:30 UTC
Actualizado el jueves, 16 de octubre de 2014 a las 08:59 UTC
Etiquetado como blog-stack, internet, en-castellano, hackers, hackinbadakigu, privacidad, deep-web, cosas-que-veo, tor, seguridad

Como muchos ya sabéis TOR es un sistema de navegación anónima del que ya hemos hablado en Laborategia en alguna ocasión. Entre otras cosas nos permite ser algo mas anónimos en la Red asi como entrar en un parte de la Internet no comercial que algunos desconocíamos hasta probar el software. Si queréis mas información sobre TOR os recomiendo leer los articulos que escribi en su dia bajo ese tag.

El tema es que últimamente al instalar Vidalia y configurar con TOR lanzaba un mensaje de error diciendo que no se podía inicial TOR, al menos en entornos Linux. Es debido a una mala gestión con AppAmor que podemos solucionar con tres simples comandos:

sudo ln -s /etc/apparmor.d/usr.bin.vidalia /etc/apparmor.d/disable/
sudo apparmor_parser -R /etc/apparmor.d/usr.bin.vidalia
sudo /etc/init.d/tor start

Una vez solucionado podremos arrancar Vidalia sin ningun tipo de problemas y se conectara de forma automática a la red TOR.

Ante este problema mucha gente esta optando por usar el Bundle de TOR que distribuyen desde la pagina del proyecto, es muy fácil de usar, simplemente desempaquetar y ejecutar ya lo tendremos en marcha. Pero si usamos este método supone un problema no poder TORificar las demás aplicaciones (mensajería instantánea, IRC…) por lo que os recomiendo solucionar el problema instalando el software desde los repositorios de Ubuntu y ejecutando los tres comandos anteriores para solucionar el error.
