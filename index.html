Jerarquías ad-hoc en Clojure

lunes, 06 de abril de 2015, fuente Koalite

A la hora de aplicar polimorfismo, en los lenguajes orientados a objetos tradicionales como Java o C#, una de las herramientas más utilizadas es la herencia entre clases o interfaces, que nos permite establecer una jerarquía entre los diferentes tipos que forman parte de una aplicación. Esto nos permite enviar mensajes (lo que suele traducirse en invocar métodos) de forma uniforme sobre distintos tipos de datos, y dejar que cada tipo interprete el mensaje de una forma específica. Este tipo de jerarquías presenta una limitación importante: se definen de forma estática al declarar las clases y no podemos modificarlas a posteriori.
